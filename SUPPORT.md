# Support

Use GitHub issues for reproducible bugs, feature requests, packaging problems, documentation fixes, and research task proposals.

Before opening an issue:

- Check the README and existing issues.
- Prepare a minimal reproducible example when reporting a bug.
- Include Python, PyTorch, CUDA, and package versions for runtime issues.
- Include dataset version, split details, and command-line arguments for experiment issues.
- Redact logs and do not include credentials, private participant data, or confidential records.

Security vulnerabilities, leaked secrets, and sensitive data concerns should be reported privately through [SECURITY.md](SECURITY.md).
